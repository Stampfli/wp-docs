genesis_save_custom_fields ( array  $data,
  $nonce_action,
  $nonce_name,
  $post,
  $deprecated = null 
)

Save post meta / custom field data for a post or page.

It verifies the nonce, then checks we're not doing autosave, ajax or a future post request. It then checks the current user's permissions, before finally* either updating the post meta, or deleting the field if the value was not truthy.

By passing an array of fields => values from the same metabox (and therefore same nonce) into the $data argument, repeated checks against the nonce, request and permissions are avoided.

Since
1.9.0
Parameters
array$dataKey/Value pairs of data to save in '_field_name' => 'value' format.
string$nonce_actionNonce action for use with wp_verify_nonce().
string$nonce_nameName of the nonce to check for permissions.
WP_Post | integer$postPost object or ID.
integer$deprecatedDeprecated (formerly accepted a post ID).
Returns
mixed Return null if permissions incorrect, doing autosave, ajax or future post, false if update or delete failed, and true on success.
